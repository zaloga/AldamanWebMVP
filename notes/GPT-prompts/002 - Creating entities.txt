1.

připrav entity pro obsahové stránky webu:

PageDefinitionEntity
Účel: katalog systémových stránek.
Pole:
•	Id 
•	PageKey — unikátní, např. home, about 
•	RouteSegment 
•	IsHomePage 
•	IsActive 
•	DefaultSortOrder 
•	CreatedAtUtc 
•	UpdatedAtUtc 
Poznámka: tohle není redakční obsah. Je to systémový registr stránek.
PageContentEntity
Účel: obsah konkrétní stránky pro konkrétní jazyk.
Pole:
•	Id 
•	PageDefinitionId 
•	LanguageCode — cs, en 
•	Title 
•	Slug 
•	SeoTitle 
•	SeoDescription 
•	SectionsJson 
•	IsPublished 
•	PublishedAtUtc 
•	CreatedAtUtc 
•	UpdatedAtUtc 
SectionsJson bude serializovaný payload sekcí stránky. Tím odpadne samostatná entita PageContentBlockEntity a výrazně klesne pracnost MVP. 
Doporučený tvar SectionsJson
Například pole objektů:
[
  {
    "type": "Hero",
    "heading": "Aldaman",
    "subheading": "Text...",
    "primaryButtonText": "Kontakt",
    "primaryButtonUrl": "/cs/kontakt",
    "backgroundMediaId": "..."
  },
  {
    "type": "RichText",
    "heading": "O mně",
    "html": "<p>...</p>"
  }
]
To je dostatečně flexibilní a stále levné.
